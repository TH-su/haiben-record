クラウド同期設定

本アプリは Googleスプレッドシート をメインの保存先、GitHub をバックアップとして使用します。 利用者ごとのジェノグラムが、複数端末で自動同期され、誤って削除しても履歴から復元できます。
① Googleスプレッドシート ＋ Apps Script の準備（必須）
  1. Googleアカウントにログイン → https://sheets.new で新規スプレッドシートを作成。シート名を「genograms」に変更。
  2. メニュー「拡張機能」→「Apps Script」を開く。
  3. 表示されたエディタの内容を全て削除し、下の「📋 GASコードをコピー」ボタンの内容を貼り付け。
  4. 左上のプロジェクト名（無題のプロジェクト）を「genogram-sync」等に変更し保存（💾）。
  5. 右上「デプロイ」→「新しいデプロイ」→歯車「種類の選択」で「ウェブアプリ」を選択。
  6. 説明：任意、次のユーザーとして実行：「自分」、アクセスできるユーザー：「全員」を選択 →「デプロイ」。
  7. 初回はアクセス許可が必要。「アクセスを承認」→ アカウント選択 →「詳細」→「（プロジェクト名）に移動」→「許可」。
  8. 表示された「ウェブアプリ」のURL（https://script.google.com/macros/s/.../exec）をコピーし、下のフォームに貼り付け。
② GitHub バックアップの準備（推奨・任意）
  1. https://github.com/new で新しいリポジトリを作成（例: genogram-data、Privateを推奨）。
  2. https://github.com/settings/tokens?type=beta で「Fine-grained personal access tokens」→「Generate new token」。
  3. Token name 任意、Expiration（有効期限）90日〜1年、Repository access「Only select repositories」で先程作ったリポジトリを選択。
  4. 「Repository permissions」→「Contents」を Read and write に設定 → 下までスクロールして「Generate token」。
  5. 表示された github_pat_... をコピー（一度しか表示されません）。
  6. 下のフォームに「オーナー名 / リポジトリ名 / ブランチ（main） / トークン」を入力。

💡 信頼性について
・保存ボタン押下時、まず端末内（localStorage）に即時保存され、その後スプシ・GitHubへ非同期送信されます。
・通信が失敗した場合は自動的にリトライキューに入り、次回保存時 / 「↻ 同期」ボタン押下時に再送されます。
・端末がオフラインでもアプリは動作し、回線復帰後に自動同期します。
・別端末で先に保存されていた場合、保存時にバージョンを比較し上書き確認します。
・トークンは端末のlocalStorageに保存されるため、共用PCでの利用後は「設定をクリア」を実施してください。

■ 操作の手順
① 左パレットで「男性/女性/不明」を追加
② キャンバス上でドラッグして配置
③ 関係線・同居枠・テキストを作成
④ 上部「保存」で利用者ごとに記録、「JPG/PDF出力」で書き出し

■ 関係線の引き方ルール
婚姻 ─: ボタン押下→ 結ぶ2人をクリック（順序不問）。
離婚 ⫽: 婚姻と同じ操作。中央に二重斜線が入る。
親子 ┃: ボタン押下→ 親→子 の順にクリック。親が婚姻線を持つ場合は夫婦の中央から自動で線が分岐する。
兄弟 ⊥: ボタン押下→ 兄弟2人をクリック。一方に親子線がある場合、その親子線の中盤レベルに兄弟バーが揃って配置される。
同居枠: ボタン押下→ 含めるノードを順にクリック→ 再度ボタンで確定。
テキスト: ボタン押下→ 配置したい場所をクリック。直後にプロパティ欄で文字入力。

モードのキャンセルは Esc、選択中の削除は Delete。
